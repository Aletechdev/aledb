ALE Analytics

This web platform has been built to support the report generation and consolidation of adaptive mutations from Experimental Evolutions and Adaptive Laboratory Evolution (ALE) experiments.

Feature Overview

The ALE Analytics web platform maintains two categories of features: those that describe individual ALE experiments, and those that describe all consolidated ALE experiment data (Figure 2B). To describe individual ALE experiments, ALE Analytics generates reports that describe the mutation lineage of ALEs, potential key mutations, and metadata reports describing the conditions for experiment isolates. To describe the consolidated ALE experiment database, ALE Analytics provides a mutation search feature, an automated ALE experiment comparison feature, and the ability to export the mutation data from one or more ALE experiments in a flat file format.

ALE Experiment

The foundational work to which this thesis builds on are the ALE experimental wet lab methodologies. Beginning with a well known starting strain, an ALE experiment is commonly executed by serially passing a selected cell culture to a fresh flask of media, enabling the particular strain passed to continue evolving under the experimental conditions. The selection of cells to pass on to the next flask is often based on growth rate since we assume that the most adapted strain population will outgrow their competition. ALE experiments can also involve replicate ALEs, which are identical experimental evolutions that execute in parallel. This approach reveals additional mutational data on the dynamics of adaptation and evolution for an organism and can describe a convergence of adaptations across multiple ALEs. ALE experiments can involve multiple ALEs with multiple flasks and isolates. In addition, each isolate can have one or more technical replicates, which are used to serialize duplicate isolates with the goal of increasing the quality of a previous isolate. The work described by this thesis serializes each sample with an abbreviation for each ALE layer and a count value representing a sample’s identity within an ALE layer’s sequence. For example, the serialization A6 F21 I2 R1 represents the first technical replicate of the second isolate from the 21st flask of the 6th ALE in an ALE experiment. The output of an ALE experiment are populations or clonal isolates which were derived from the original strain and additionally include the mutations in the cells according to the environmental stresses introduced in the ALE experiment. This output is then processed by additional foundational technologies leveraged with ALE: the DNA sequencer and mutation identifying software. The output sample to be analyzed is sequenced, typically by an Illumina platform, and resequenced primarily using the Breseq computational pipeline. Breseq’s resequencing process aligns all of the DNA reads to a provided genome reference and reports single-nucleotide mutations, point insertions and deletions, large deletions, missing coverage and new junctions according to the differences between the aligned reads and the reference genome.

Generally, researchers compare the mutations from each ALE’s endpoint samples to identify genomic regions with many alleles. If a researcher sequenced ALE midpoint or intermediate samples, an ALE’s mutations can be organized in chronological order to identify mutations that fix. The mutations that are involved in multiple alleles of a genomic region or are shown to fix from starting or midpoint to endpoint samples are considered the key mutation sets and are investigated for their possible fitness benefit according to the genomic region they affect.

Mutation tables

The primary focus of the ALE Analytics web platform is ALE mutation report generation. Each ALE experiment can be described as a series of isolate mutation sets in relation to an ALE’s starting strain. Ordering sample mutation sets as columns from earliest to latest in an ALE serves as a visualization to grant intuition on mutational trends. The occurrence of a mutation in a sample is annotated with an allele frequency within the cell of a mutation row for the sample.

Above is an example mutation table. This mutation table represents an ALE lineage report, where samples are represented as columns, ordered from left to right as earliest to latest in an ALE. Rows describe the specific mutations manifested within the sample set, and values contained within cells represent the allele frequency.This format enables researchers to intuitively identify important mutational patterns, such as the fixed mutations with corA and the hns/tdk intergenic region.

Mutation lineage reports enable researchers to recognize patterns of key adaptive mutations. Among the many mutations that manifest within an ALE experiment, mutation rows that describe multiple alleles of a gene will cluster together with the sorting of mutations according to their positions on the genome. This is illustrated with the mutated corA within the example table. Due to the chronological sorting of the sample columns per ALE, a mutation that fixes across samples will manifest as an unbroken sequence of cells in a mutation row annotated with an allele frequency. This is illustrated with both the hns/tdk and corA mutations in the example table. These two patterns are obvious to an observer and serve well to describe the mutational trends in ALE experiments.

Key mutation reporting

Using the ALE Analytics mutation reporting mechanism, we have implemented features that automate the identification and reporting of mutations which describe significant mutational patterns within an ALE experiment. These features describe key and potentially adaptive mutation patterns which we refer to as enrichment and fixed mutations. The methods encapsulated in the enrichment and fixed mutation analysis are those which have been developed to manually identify adaptive mutations within the results of published ALE experiments (PGI) (C13) (42C) (GLU) and are therefore considered common ALE experiment key mutation analysis. Once an experiment’s initial set of key mutations are identified, additional validation experiments and functional analysis are executed to validate that they are adaptive. As with the manual consolidation of ALE experiment mutation data, the analysis of adaptive mutations can be prone to human error, inconsistent between researchers and time consuming. The automation of these common analyses will contribute to more accurate results, more consistent analysis and the shortening of turnaround time from ALE experiment execution to results interpretation.

In the SBRG’s published ALE experiment literature, adaptive mutations were identified as those involved in two cases. If a mutation manifested in more than one sample, it signified that this mutation described a dominant phenotype. If a genomic region was mutated via different mutations across multiple samples, it was hypothesized that simply perturbing the gene may have rendered a beneficial phenotype. Considering that these circumstances all involve isolates with the same mutated genomic region, we can also consider a single sample with multiple alleles of the same region to represent multiple populations which benefit from this region’s perturbation. The identification of these cases can be accomplished by finding genomic regions within an ALE experiment that have more than one observed mutation; we consider these enriched genomic regions and their mutations as enrichment mutations. Previous to ALE Analytics, this analysis was accomplished as additional steps in manually curating the ALE experiment’s mutation report such that mutations affecting the same genomic region would cluster together within a matrix of mutations and their samples. ALE Analytics automates this enrichment mutation analysis and reports the results in the same mutation table format.

A fixed mutation is one in which a mutation manifests in an ALE’s midpoint, or intermediate sample, and is propagated to all following samples in the ALE. This analysis is only possible if an ALE experiment includes midpoint samples, providing the possibility of more than one data point per mutation. Previous to ALE Analytics, the identification of fixed mutations was accomplished as an additional results reporting task by manually organizing mutations according to their ALE’s sample chronology and identifying mutations that emerge in a midpoint and manifest in all remaining ALE samples. The fixed mutation analysis automates this approach and reports results in the mutation table format.
